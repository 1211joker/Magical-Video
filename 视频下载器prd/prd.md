# Product Requirements Document: AI 万能视频下载总结器

**版本**: v1.0
**日期**: 2026-06-27
**状态**: 初稿
**作者**: 独立开发者

---

## 1. Executive Summary

AI 万能视频下载总结器是一个基于 yt-dlp + DeepSeek AI 的 Web 应用，让用户在下载视频 **前** 先了解视频核心内容。核心场景：粘贴链接 → AI 自动提取字幕 → 生成摘要/要点/思维导图 → **用户判断值不值得看** → 决定是否下载。MVP 阶段支持 YouTube 和 B 站。

> **一句话 MVP**：在不看的提前下，知道视频讲了什么；在知道值不值得后，一键下载到本地。

## 2. Background & Context

### 问题空间

作为普通消费者，很多人都有下载保存视频到本地的需求。但现有工作流存在两大痛点：

1. **信息不透明**：看到一个 1 小时的视频，不点开不知道讲什么，点了又浪费时间
2. **下载决策盲目**：下载才发现内容不对胃口，浪费带宽和存储空间

现有方案对比：

| 方案 | AI 预览 | 多平台支持 | 下载功能 | 费用 |
|------|---------|-----------|---------|------|
| 直接 yt-dlp 命令行 | ❌ | ✅ | ✅ | 免费 |
| 在线下载站 | ❌ | ❌ | ✅ | 含广告 |
| 浏览器插件 | ❌ | 仅 YouTube | ✅ | 不等 |
| **本产品** | ✅ | ✅ (YTB + B站) | ✅ | MVP 免费，后续出付费 |

### 技术发现

通过真实环境技术验证（技术 Spike），关键结论：
- **YouTube**：元数据提取 ✅，字幕提取需处理 PO token（yt-dlp 社区已知问题 #12482）
- **B站**：元数据 ✅，字幕 ✅（需从 Chrome 读取 cookies）
- **抖音/TikTok**：❌ 不可用（TikTok 提取失败，抖音 Bug #9667 长期未修复）

### 竞品格局

> 完整竞品分析见独立文档 `competitor-analysis.md`

AI 视频摘要赛道已形成清晰格局，其中 **BibiGPT（100万+用户）** 和 **NoteGPT（40万+ Chrome 安装）** 是直接竞品。

| 维度 | BibiGPT | NoteGPT | 本产品差异化 |
|------|---------|---------|------------|
| **定位** | AI 音视频助理 | AI 学习助手 | **摘要 + 下载一站式** |
| **视频下载** | ❌ 不支持 | ❌ 不支持 | **✅ 核心差异点** |
| **月费起价** | $19.80 | $9.99 | **MVP 免费，后续更低** |
| **核心场景** | 理解视频内容 | 学习笔记整理 | **判断是否值得下载并保存** |

**关键洞察**：两个竞品都只做"摘要"不做"下载"，用户看完摘要想下载还需要另找工具。这是我们最大的差异化空间。详见 `competitor-analysis.md:5. 差异化定位分析`。

## 3. Objectives & Success Metrics

### 目标

1. **摘要先行**：用户在下载前，通过 10 秒阅读就能判断视频值不值得看
2. **一键下发**：觉得值得看后，一次点击就能下载到本地
3. **知识结构化**：以核心要点 + 思维导图的形式呈现视频知识框架

> **主路径**：粘贴链接 → 看摘要 → **决定** → 下载。下载不是并列功能，而是摘要决策后的出口。

### 非目标（MVP 不做的）

1. **抖音/TikTok 支持** — 等 yt-dlp 社区修复后再评估
2. **Whisper 语音转文字** — MVP 仅做有字幕视频，无字幕兜底放 Phase 2
3. **付费功能** — MVP 完全免费，后续再规划增值功能
4. **本地客户端** — MVP 保持纯 Web 架构
5. **用户系统、历史记录、云端存储** — 不做

### 成功指标（含测试条件）

**测试前提**：以下指标均在以下环境测得
- 网络：100Mbps 宽带，到 YouTube/B站 延迟 <100ms
- 视频类型：时长 5-60 分钟，语言为中文或英文，有平台原生字幕
- 设备：Chrome 最新版，macOS/Windows 桌面端

| 指标 | 目标 | 测量方式 | 降级判定 |
|------|------|---------|---------|
| AI 摘要生成成功率 | ≥ 95% | 分母=所有有字幕视频请求，分子=成功返回完整摘要的请求 | 失败时展示原始字幕文本 |
| 粘贴到摘要展示（端到端） | P50 ≤ 15 秒，P95 ≤ 30 秒 | 从用户点击"AI分析"到页面渲染出摘要的时间戳差 | 超过 30 秒显示超时提示，可重试 |
| 元数据展示（粘贴到信息展示） | P95 ≤ 5 秒 | 从粘贴 URL 到缩略图/标题渲染 | 5 秒无响应展示骨架屏 |
| 用户完成率（走完"摘要→下载"全流程） | ≥ 70% | （点赞摘要按钮 + 发起下载数）/ 总分析完成数 | — |
| 思维导图生成成功率 | ≥ 95% | 有摘要后成功渲染 markmap 比例 | 导图失败仅展示文字摘要 |

## 4. Target Users & Segments

### 一级用户（MVP 核心）

**知识型学习者**
- 频繁在 YouTube/B站 学习教程、讲座、技术分享
- 痛点：长视频"信息密度不明"，想快速判断值不值得看
- 典型场景：收藏了 50+ 视频但一直没看，希望 AI 提炼核心内容
- 技术能力：会用浏览器，不熟悉命令行

### 二级用户（后续覆盖）

**视频收集爱好者**
- 习惯把喜欢的视频下载到本地保存，担心在线资源失效
- 痛点：下载前想确认视频质量（内容层面）
- 当前工作流：手动看评论区、看进度条跳着预览

## 6. Solution Overview

### 系统架构

```
┌─────────────────────┐
│     用户浏览器       │
│  Vue 3 + Vite SPA    │
│  - 链接输入 + 识别    │
│  - 元数据展示         │
│  - AI 摘要 + 导图     │
│  - 下载管理           │
└─────────┬───────────┘
          │ HTTP / SSE
          ▼
┌─────────────────────┐
│   FastAPI 后端       │
│  - yt-dlp 封装       │
│  - DeepSeek AI 调用  │
│  - SSE 进度推送      │
│  - 文件代理下载       │
└─────────────────────┘
```

### 关键设计决策

| 决策 | 方案 | 理由 |
|------|------|------|
| **AI 模型** | DeepSeek API | ¥0.01/次 vs GPT 的 ¥0.05+，成本极低 |
| **字幕获取** | yt-dlp 直接提取 | 无需下载完整视频，速度快 |
| **分析进度** | SSE 实时推送 | 避免用户因等待而流失 |
| **思维导图** | markmap 纯前端渲染 | 不占服务器资源，导出方便 |
| **视频下载** | 服务器代理流式转发 | 隐藏真实源，不落地到服务器 |
| **YouTube PO token** | 初期使用 `--extractor-args` 传空 token，依赖 yt-dlp 社区解决方案；若字幕获取失败则降级为"本视频暂不支持 AI 摘要"，仍可下载 | 不阻塞 MVP，PO token 是社区已知问题且有修复预期 |
| **B站 cookies** | 使用 `--cookies-from-browser chrome` 自动读取服务端 Chrome 配置（如部署环境有 Chrome）；兜底方案为辅图指引用户手动导出 cookies 文件上传 | 优先自动化，不行再手动 |

### 用户流程

```
粘贴链接 → 自动识别平台 → 展示视频信息
                              │
                     ┌────────┴────────┐
                     ▼                 ▼
                [AI 摘要]          [立即下载]
                     │
              SSE 实时推送进度
                     │
           ┌─────────┼─────────┐
           ▼         ▼         ▼
        文字摘要   核心要点   思维导图
```

### AI 摘要格式规范

DeepSeek 输出结构（JSON）：

```json
{
  "title": "视频标题",
  "summary": "2-3 句话概括视频核心内容...",
  "key_points": [
    "要点 1：...",
    "要点 2：...",
    "要点 3：..."
  ],
  "mindmap": {
    "root": "视频主题",
    "children": [
      {"name": "章节 1", "children": [{"name": "子点"}]},
      {"name": "章节 2", "children": []}
    ]
  }
}
```

> **Prompt 策略**：将字幕分段后，提示 DeepSeek 按"核心摘要 + 关键要点 + 知识结构"三层输出。

## 7. 安全、隐私与运营

### 数据安全

| 问题 | 处理方式 |
|------|---------|
| **视频链接** | 仅用于本次请求，不存储到数据库或日志 |
| **B站 cookies** | 仅在内存中使用（传给 yt-dlp 进程），不持久化存储；随请求结束释放 |
| **字幕内容** | 发送给 DeepSeek API 做分析，不在服务器留存；DeepSeek 官方承诺不用于训练 |
| **下载视频** | 服务器实时流式转发，**不落地到磁盘**，转发完毕即释放 |
| **用户 IP** | 不在服务端记录用户 IP，仅 Nginx 层面保留基础访问日志（7 天轮换） |

### 版权与合规

- **免责声明**：首页/关于页面展示"本工具仅用于个人学习和合理使用，请遵守各平台服务条款"
- **不提供盗版功能**：不破解 DRM、不绕过付费墙、不存储受版权保护内容
- **平台条款**：符合 YouTube ToS 合理使用范围、B站 用户协议

### 运营控制与成本

| 措施 | 详情 |
|------|------|
| **AI 请求限流** | 单 IP 每分钟最多 10 次 AI 分析请求；每次分析间隔 ≥ 3 秒 |
| **下载限速** | 单次下载最高 50Mbps，避免带宽被打满 |
| **文件大小限制** | MVP 限制单文件 ≤ 2GB（超过提示"付费版支持"） |
| **并发限制** | 同一用户同时最多 2 个任务（分析或下载） |
| **成本预估** | MVP 假设日活 100 人、人均 3 次分析、人均 1 次下载（500MB）→ 日带宽 ~50GB，日 AI 成本 ~¥3 |

## 附录 A：技术选型原则

> PRD 层面记录"为什么选这个、选型约束条件"。具体 pinned 版本号在开发计划/技术方案文档中锁定。

| 选型 | 选它而非其他的理由 | 约束条件 |
|------|-------------------|---------|
| **FastAPI** (Python) | Python 是 yt-dlp 原生生态；FastAPI async 原生支持 SSE；部署简单 | Python ≥ 3.11 |
| **yt-dlp** | 社区最活跃的下载引擎；多平台支持；纯 CLI 容易封装 | 必须 nightly 版本以支持 impersonate；需要 curl_cffi |
| **DeepSeek API** | 价格仅为 GPT-4o 的 1/20；API 原生支持 JSON mode；国内延迟低 | 需要 API key；有并发限制 |
| **Vue 3 + Vite** | 轻量；CLAUDE.md 已有推荐；适合中小型 SPA | 无特殊约束 |
| **markmap** | 纯前端无需服务端渲染；支持交互缩放和导出；MIT 协议 | 依赖 markmap-lib + markmap-view 包 |
| **SSE** | 比 WebSocket 轻量；单向推送够用；浏览器原生 EventSource 支持 | 需要反向代理（如 Nginx）正确配置 buffer |
| **Shadowsocks** | 开发环境解决 YouTube 访问问题 | 仅在开发/服务端配置，不影响前端 |

| 风险 | 可能性 | 影响 | 缓解方案 |
|------|--------|------|---------|
| YouTube 反爬升级，字幕获取失效 | 中 | 高 | 准备降级文案"本视频暂不支持 AI 摘要"；跟踪 yt-dlp 每日更新 |
| B站 cookies 失效 | 中 | 中 | 提供自动检测 + 重新导入指引；准备自动化方案（服务器 Chrome） |
| DeepSeek API 超时/限流 | 低 | 中 | SSE 告知用户排队中；20s 超时降级显示原始字幕；前端限速（单 IP 10次/分） |
| 服务器带宽成本超预算 | 低 | 中 | MVP 日 50GB 以内可承受；超出后考虑限速或付费版 |
