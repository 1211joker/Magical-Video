# Pre-mortem: AI 万能视频下载总结器

> **方法**：假设产品已经上线并失败了，回头看"为什么会死"。从技术、用户、运营、外部依赖四个角度逐一推演。

---

## 🔴 致命级（直接导致产品不可用，必须提前防御）

### 1. YouTube 反爬升级，字幕获取全部失效

**推演**：上线后第 3 周，YouTube 更新了页面结构或签名算法，yt-dlp 社区 48 小时内未出修复。所有 YouTube 视频的摘要功能瘫痪，用户只能下载不能预览。

**根因**：产品严重依赖 yt-dlp 的逆向工程能力，对 YouTube 变化零控制力。

**缓解措施**：
- 🟢 架构上把字幕提取层与 AI 摘要层解耦 → YouTube 挂了至少还能做 B站
- 🟢 监控 yt-dlp 版本更新，收到 release 后自动/手动升级
- 🟢 准备应急方案：提醒用户"本视频暂无字幕，可尝试下载后用本地播放器查看"
- 🟡 长期：Whisper 语音转文字完成时不依赖平台字幕

### 2. B站 cookies 过期，用户无法获取任何信息

**推演**：用户按指引导出 cookies，但 3 天后 cookies 过期，B站所有功能（元数据+字幕）都返回 412/403。用户反复操作 cookies 后放弃。

**根因**：B站依赖浏览器 cookies 认证，而 cookies 有有效期。

**缓解措施**：
- 🟢 UI 上明确提示"B站需要登录态，每 7 天更新一次"
- 🟢 提供一键式的 cookies 导出指引（图文步骤）
- 🟢 检测到 cookies 失效时，给出明确错误提示而非空白页
- 🟡 探索更稳定的认证方案（如 OAuth）

### 3. DeepSeek API 超时/降级/限流

**推演**：用户高峰时段，DeepSeek API 响应超过 30 秒或返回 429（限流）。摘要生成失败，用户只看到空白。

**根因**：第三方 API 没有 SLA 保障。

**缓解措施**：
- 🟢 SSE 推送真实进度，卡住时告知"AI 分析排队中，请稍候"
- 🟢 设置 20 秒超时，超时后降级为"显示原始字幕"（有字幕总比没有好）
- 🟢 后端限流：个人用户每分钟最多调用 5 次 AI 分析

---

## 🟡 严重级（影响体验和口碑，但可修复）

### 4. 视频下载太慢，用户中途放弃

**推演**：服务器带宽只有 10Mbps，下载一个 2GB 视频需要 30 分钟。用户等不及关闭页面，再也不用。

**根因**：未对下载做流式优化和速度预期管理。

**缓解措施**：
- 🟢 下载前预估文件大小和预计时间，展示进度条
- 🟢 支持断点续传（HTTP Range Header）
- 🟢 后端下载使用流式传输，不缓存到服务器
- 🟡 MVP 上线后监控带宽使用率，决定是否升级服务器

### 5. 思维导图对长视频内容太拥挤

**推演**：1 小时的教学视频有 20+ 章节，markmap 渲染出密密麻麻的导图，用户无法阅读。

**根因**：未考虑内容量对可视化的影响。

**缓解措施**：
- 🟢 markmap 本身支持缩放和折叠，默认折叠子节点
- 🟢 限制导图层级最大 3 层
- 🟢 提供"仅显示核心节点"精简模式
- 🟢 支持截图保存时自动缩放适配

### 6. 非技术用户不会用，转化率低

**推演**：目标用户是"普通消费者"，但他们看不懂"在 Chrome 地址栏输入 chrome://version 找到 Profile Path 然后导出 cookies"这类操作。B站功能实际使用率极低。

**根因**：高估了非技术用户的操作能力。

**缓解措施**：
- 🟢 B站 cookies 指引做到极致：图文并茂、每个步骤截图
- 🟢 首页默认放一个 YouTube 视频的完整演示（GIF 或录屏）
- 🟢 如果 `--cookies-from-browser` 可以在服务端读取 Chrome 配置（如果部署环境有浏览器），则完全自动化

---

## 🟢 轻微级（可接受，上线后迭代）

### 7. AI 摘要质量不稳定

**推演**：有些视频的字幕质量差（自动字幕、机翻、杂乱无格式），DeepSeek 摘要跑偏。

**评估**：可以接受。告知用户"AI 摘要仅供参考，基于字幕生成"即可。不承诺 100% 准确。

### 8. 用户粘贴非视频链接

**推演**：用户粘贴普通网页、图片链接等，系统返回"不支持的链接"。

**评估**：正常。给出明确提示，引导用户贴正确的链接格式。

### 9. 被滥用为侵权工具

**推演**：有人批量下载有版权保护的视频内容。

**评估**：确实需要重视。
- 🟢 首页/关于页面添加版权声明："请遵守各平台服务条款，仅用于个人学习和合理使用"
- 🟢 不提供批量下载功能（MVP 限制单次处理）
- 🟢 不存储任何视频文件到服务器

---

## Pre-mortem 总结

| 等级 | 风险数 | 必须处理 |
|------|--------|---------|
| 🔴 致命 | 3 | **YouTube 反爬、B站 cookies、DeepSeek 降级** |
| 🟡 严重 | 3 | 下载速度、导图拥挤、非技术用户门槛 |
| 🟢 轻微 | 3 | 可接受，上线后迭代 |

**最大的三条死因**：
1. 过度依赖 yt-dlp 和第三方平台 → 多做一层降级兜底
2. B站的认证门槛把非技术用户挡在门外 → 简化 cookies 获取流程
3. 后端带宽/API 超时导致体验差 → 提前做好限流和降级

> **核心行动项**：从开发第一天就把"降级方案"写在代码里——字幕失败时怎么显示、API 超时怎么显示、cookies 过期怎么提示。这些不应该是上线后补的。
